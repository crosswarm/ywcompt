# 评审记录

## 第一轮：时间语义

结论：字段命名采用 `receivedAt`，但文档明确它是“最佳可得的任务到达时间”，不是无条件精确的 assignment event。

调整：禁止使用提交时间、同步时间或多个时间的最大值兜底；把来源和语义作为数据契约的一部分。

## 第二轮：获取效率

结论：不在同步中逐条调用 `todo-detail`。

调整：在 `list-inbox` 内按当前租户 assignee 去重批量查 workflow task，再按 task id 合并；跨租户直接走消息中心降级。

## 第三轮：失败与历史状态

结论：workflow 批量查询失败不能让已知强来源在下次同步中退化。

调整：同步合并同一 task id 的历史 `receivedAt` 元数据；新 task id 不继承。

## 第四轮：UI 诚实性

结论：无到手时间时不显示提交时间冒充。

调整：列表空值展示“-”且排序置后；详情独立展示到手时间、来源和提交时间。消息中心来源明确标记“近似”。

## Gate Result

用户明确要求“按照建议进行”，并要求明确取值规则、来源和降级方案。视为已通过实现 Gate。
