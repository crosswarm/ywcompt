# 智能待办 CLI 启动目录与能力门禁设计

## 运行时边界

- `bip-cli-client.mjs` 统一执行 `--schema` 与业务命令，默认 `cwd` 为 `bip-cli.js` 所在目录；CLI 路径和显式 `cwd` 必须是绝对路径，避免再次依赖失效的父进程目录。
- `--schema` 返回数组中的 `path` 是唯一兼容性依据；版本号不参与判断。
- 每次业务调用前校验目标命令。能力集合按 CLI 路径、文件大小和修改时间缓存，文件变化后重新探测。
- 智能待办完整依赖能力集中为一个十项常量，供运行时和打包脚本共同使用。

## 发布边界

- `pack-skill.mjs` 在删除旧目录或 ZIP 前完成十项能力校验。
- 任一校验失败时进程非零退出，旧产物保持原样。
- 校验通过后继续使用既有过滤规则生成目录和 ZIP，不改变发布包结构。

## 非目标

- 不修改本地 HTTP 接口、审批数据和业务分流。
- 不修改 `bip-cli` 仓库，不新增旧命令或直连 HTTP 降级。
- 不使用 `BIP_PLATFORM_VERSION` 规避 `uv_cwd`。
